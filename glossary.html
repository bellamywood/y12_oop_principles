AS Level OOP Glossary

This glossary contains Tier 2 (academic) and Tier 3 (technical) vocabulary used in OCR-style exam questions and this workbook.

Tier 3 – Technical OOP Vocabulary

Attribute
A piece of data stored inside an object that represents its state.
Exam use: “Identify an attribute of the Prize class.”
Class
A blueprint that defines the attributes and methods that its objects will have.
Exam use: “Define a class in pseudocode.”
Constructor
A special method used to initialise an object when it is created.
Exam use: “Write the constructor for class Treasure.”
Encapsulation
Restricting direct access to an object’s data by making attributes private and providing controlled access through methods.
Exam use: “Describe how encapsulation is used in this class.”
Inheritance
A mechanism where one class (subclass) derives attributes and methods from another class (superclass).
Exam use: “Explain how inheritance reduces duplication.”
Instantiation
The process of creating an object from a class.
Exam use: “Instantiate a new Prize object.”
Method
A function defined within a class that performs an action.
Exam use: “State what the method getValue() does.”
Object
An instance of a class that contains data and behaviour.
Exam use: “What is an object in this context?”
Overriding
When a subclass provides its own version of a method defined in its superclass.
Exam use: “Explain how overriding supports polymorphism.”
Polymorphism
Different object types responding differently to the same method call.
Exam use: “Describe polymorphism in this example.”
Procedure
A named block of code that performs a specific task (often in pseudocode).
Exam use: “Write a procedure guessGrid(row,column).”
Return Value
The value sent back from a method to the caller.
Exam use: “What is the return value of getLevel()?”
Selection
A control structure (if/else) that chooses actions based on conditions.
Exam use: “Identify the selection in updateValues().”
Sequence
The order in which instructions are executed.
Exam use: “Explain the sequence of steps.”
Iteration
Repetition of a block of code using a loop.
Exam use: “Explain how iteration is used.”
Array
A fixed-size indexed data structure storing elements.
Exam use: “Why is an array suitable?”
Index
The numeric position used to access an element in an array.
Exam use: “What does index 3 represent?”

Tier 2 – Academic Vocabulary

Analyse
Examine in detail to explain or interpret components.
Concept
An abstract idea or principle.
Criteria
Standards used for judgement or marking.
Interpret
Explain the meaning of something.
Process
A series of actions taken to achieve a result.
Significant
Important or meaningful.
Structure
The way parts are organised or arranged.
