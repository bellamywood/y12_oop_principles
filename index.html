0) What is OOP and why does industry use it? Start Here

Metaphor: Building a game by writing one giant list of instructions is like building a city with no street names. OOP adds structure: you create blueprints (classes) and build things (objects) that know their own data and behaviour.
What OOP is (OCR-friendly):
Object-oriented programming is a paradigm where a program is structured using classes and objects. Each object bundles data (attributes) and behaviour (methods).
Why industry cares (practical reasons):
  • Maintainability: real software changes constantly. OOP helps teams change one part without breaking everything else.
  • Reusability: common features can be reused across multiple parts of a system (reduces duplicated code).
  • Teamwork at scale: developers can work on separate classes/modules with clearer boundaries.
  • Modelling real systems: many domains map naturally to objects (users, accounts, orders, characters, prizes).
OCR translation: industry uses OOP because it improves maintainability, reduces duplication, and supports complex systems.

Game Factory: A class is a blueprint. An object is a character built from the blueprint.
Attributes = what it has. Methods = what it can do.
Quick definitions:
Class: defines attributes/methods. Object: an instance created from a class.

2) Encapsulation = Locked Backpack Concept

Locked backpack: The character’s money/XP is inside a backpack. Other code can’t grab it directly. It must use a zip/slot like getMoney().
OCR-friendly definition: Encapsulation restricts direct access to attributes by declaring them private and providing controlled access through methods.
[2]

OCR-style: “Describe an object-oriented technique being used.” The class diagram shows private attributes and get... methods.

Encapsulation is being used because the attributes are declared as private. The data is accessed in a controlled way using methods such as get methods rather than direct access to the attributes.

3) Inheritance = Character DNA Concept

DNA: A base blueprint (Character) passes shared features to new blueprints (Wizard/Warrior). They reuse what’s shared, then add what’s unique.
OCR-friendly definition: Inheritance allows a subclass to derive attributes/methods from a superclass, improving code reuse and reducing duplication.
[2]

OCR-style: Explain how inheritance could be used in a game with classes Character, Player, and Enemy.

Player and Enemy could be subclasses of Character and inherit shared attributes and methods such as name, position and move(). This reduces duplicated code and makes the program easier to maintain because shared changes are made once in Character.

4) Polymorphism = Same Button, Different Result Concept

Same button: You press “attack” for every character, but each character attacks differently. Same message, different result.
OCR-friendly definition: Polymorphism allows different classes to implement methods with the same name but different behaviour.

5) OCR Question Types (Paper-style) Exam Practice + Self Mark

(a) Explain why an array is suitable to represent the road.
Context: road has 50 spaces (positions 0–49) and each space stores null or a Prize.

Mark it yourself target: 3 marks

An array is suitable because the road has a fixed number of spaces (50). Each road space has a numbered position so the position can be used as the array index (0–49). This allows direct access to any space to check or store a prize/null efficiently.
[3]

Declare class Treasure (attributes + constructor only).
Attributes: private value : integer, private level : string. Constructor assigns parameters to attributes.

Mark it yourself target: 3 marks

class Treasure
    private value : integer
    private level : string

    public procedure new(v : integer, l : string)
        value = v
        level = l
    endprocedure
endclass

Write updateValues(type, value)
If type is “money” add to money. If “experience” add to experience. Otherwise no change.

Mark it yourself target: 4 marks

public procedure updateValues(t : string, v : integer)
    if t = "money" then
        money = money + v
    else if t = "experience" then
        experience = experience + v
    end if
endprocedure

Describe decisions in a random placement algorithm
Each road space can only store one prize.

Mark it yourself target: 3 marks

The algorithm must decide whether a randomly generated position is empty (null) or already contains a prize. If empty, it stores the prize and moves on; if not, it must repeat (generate another random position) until an empty space is found. This uses selection (if/else) and iteration (loop).
[2]

Describe two benefits of using OOP rather than procedural

Mark it yourself target: 2 marks

Code reuse: classes can be reused and extended (e.g., via inheritance), reducing duplicated code. Maintainability: encapsulation keeps data private and accessed through methods, reducing unintended changes and making the program easier to modify.
